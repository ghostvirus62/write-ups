Blind XSS Exploitation

CTF Write-up - Detailed Steps

Introduction

This write-up covers how I exploited a Blind XSS vulnerability in a web application to retrieve a flag. Blind XSS occurs when the payload is not executed immediately but is triggered later when an admin or other user accesses the content.

Steps to Exploit Blind XSS:

  1. Identify Vulnerable Input Field: During testing, I found an input field where user feedback was submitted. This field was not sanitized, making it a potential candidate for XSS payload injection.
  2. Inject Payload: I crafted a simple payload to test the vulnerability: 
    <script>alert('XSS')</script>
  3. Test the Payload: After injecting the payload into the input field, I submitted feedback and waited for it to be reviewed.
  4. Trigger Payload: Later, an admin accessed the feedback page, triggering the payload and executing the XSS. This allowed me to interact with the application and retrieve sensitive information.
  5. Retrieve Flag: After the admin triggered the payload, I used JavaScript to send the flag to my remote server via a callback function.

Conclusion

By exploiting the Blind XSS vulnerability, I was able to retrieve the flag by executing a payload that triggered in the admin's browser. This type of vulnerability can be difficult to detect without proper input sanitization and output encoding.
